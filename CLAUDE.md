# 원소 타일 생성기 — 프로젝트 안내서

> 이 문서는 **개발자가 아닌 분**(예: 동료 선생님, 학생, 협업자)이 읽어도
> 무엇을 만들었고 어떻게 만들어졌는지 알 수 있도록 작성되었습니다.
> 특히 **생성형 AI(Claude)를 어떤 식으로 활용했는지**는 별도 표시
> (🤖 표시)로 단계마다 정리해두었습니다.

---

## 1. 한 줄 요약

중학교 화학 수업에서 학생들이 손으로 쥐고 배울 수 있는
**원소 타일(가로·세로 8cm 정도의 정사각형 플레이트)**을
3D 프린터로 한꺼번에 여러 개 출력할 수 있도록 도와주는 프로그램.

---

## 2. 누가 / 어디서 / 왜 쓰는가

- **누가** — 중학교 과학(화학 단원) 담당 선생님, 메이커 동아리 학생
- **어디서** — 학교 컴퓨터실 + 3D 프린터가 있는 환경
- **왜** — 교과서 속 주기율표를 손으로 만질 수 있는 실물 교구로 만들어
  학생들이 **원소 기호 / 한글명 / 원자번호 / 원자량**을 직접 비교하고
  옆 타일과 끼워 맞추며 익히게 하기 위함

---

## 3. 결과물 미리보기

타일 한 장에는 다음 4가지가 도드라지게(양각으로) 새겨집니다.

```
┌──────────────────────┐
│ 52                   │  ← 원자번호 (좌측 상단)
│                      │
│        Te            │  ← 원소 기호 (가운데, 가장 큼)
│                      │
│      텔루륨          │  ← 한글 이름
│      127.6           │  ← 원자량
└──────────────────────┘
   ↑                ↑
   └─ 좌우 슬롯으로 옆 타일과 끼움
```

좌우에 작은 홈이 있어 별도의 **끼움쇠(connector)** 로 옆 타일과 결합됩니다.
3D 프린터로 인쇄하면 학생들이 직접 만지고, 분류하고, 줄 맞춰 끼울 수 있는
교구가 됩니다.

## 4. 사용자가 보게 될 화면 (GUI)

`타일생성기.bat` 파일을 더블클릭하면 다음과 같은 창이 뜹니다.

- **헤더** — "⚛ 원소 타일 생성기" (짙은 청남색 띠, 화학 교과서 느낌)
- **탭 4개**
  1. **원소 선택** — 화면 가운데 진짜 주기율표가 보이고, 원소를 클릭하면
     선택됨. 카테고리(알칼리 금속, 비금속, 할로젠, 비활성 기체 등)별로
     색이 다름. 란타넘족·악티늄족도 따로 표시.
  2. **글자 디자인** — 폰트, 굵기, 글자 크기(mm), 새김 깊이(mm)
  3. **인쇄 설정** — 타일 크기, 두께, 옆 타일과의 끼움 여유
  4. **출력 설정** — OpenSCAD 위치, 결과 파일 폴더
- **하단** — 커다란 "🧪 STL 생성 시작" 버튼 + 진행률 막대

비개발자(예: 화학 선생님)가 **검은 명령창을 한 번도 만지지 않고도**
원소 타일을 뽑을 수 있도록 설계했습니다.

## 7. 사용된 기술 / 도구

비개발자용 풀어 쓴 설명입니다.

- **Python** — 프로그램의 머리 역할. 화면을 띄우고, 사용자 입력을 모으고,
  OpenSCAD에게 무엇을 그릴지 명령합니다.
- **tkinter** — Python에 기본 포함된 GUI 도구. 별도 설치 필요 없음.
  덕분에 사용자는 Python만 깔면 모든 화면이 바로 동작합니다.
- **OpenSCAD** — 글자나 도형을 코드로 묘사하면 진짜 3D 모델로 만들어 주는
  무료 프로그램. STL(3D 프린터가 읽는 표준 형식) 파일 출력을 담당.
- **나눔고딕 / 프리텐다드 등 한글 글꼴** — 한글이 또렷이 새겨지려면 필요.
  모두 **상업적 이용이 가능한 무료 폰트**만 GUI에 노출했습니다.

## 8. 개발 중 만났던 어려움과 해결 (핵심 함정)

이 프로젝트가 단순해 보여도, 실제로는 **Windows + 한글 + OpenSCAD** 조합에서
함정이 두 군데 있었습니다. 이 두 함정을 우회한 방식이 코드의 핵심입니다.

### 함정 ①: 한글이 두부(☒☒☒)로 나옴
- **증상** — OpenSCAD에 `-D name="텔루륨"` 같은 식으로 한글을 넘기면,
  STL에 글자 자리에 빈 사각형(글리프 누락 표시)만 나옴. 영문/숫자는 멀쩡함.
- **원인** — Windows OpenSCAD가 명령줄 인자를 시스템 코드페이지(CP949)로
  해석해서, UTF-8로 보낸 한글이 망가진 채 폰트 검색에 들어감.
- **해결** — 한글을 **명령줄로 절대 보내지 않음**.
  대신 Python이 임시로 **UTF-8 SCAD 파일**을 하나 만들고, 그 안에
  변수 정의(`name = "텔루륨";`)를 적은 뒤, 그 파일을 OpenSCAD에 입력으로 줌.

### 함정 ②: 한글 폴더 경로에서 `include` 가 실패
- **증상** — `include <G:/내 드라이브/.../element_tiles.scad>` 처럼 경로에
  한글이 들어가면 OpenSCAD가 "파일을 못 연다"며 멈춤.
- **원인** — OpenSCAD의 파일시스템 라이브러리가 Windows에서 비-ASCII
  경로를 제대로 못 읽음.
- **해결** — Python이 매 실행 시 **영문만 들어 있는 임시 폴더**(예:
  `C:\Users\...\Temp\element_tile_xxxx\`) 에 SCAD 템플릿을 복사해 두고,
  그 폴더 안에서 `include <element_tiles.scad>` (파일명만, 경로 없이) 사용.

### 그 외 잡다한 함정
- **Google Drive 동기화 폴더의 일시적 락** — 폴더를 막 지우고 다시 만들면
  잠깐 "권한 거부" 에러가 남. 잠시 후엔 풀림.
- **사용자가 폴더를 잘못 들어가서 명령 실행** —
  `element-symbol-generator` 와 그 안의 `maru-python-version` 을 혼동.
  GUI 도입으로 이 문제는 사실상 해결됨.

## 9. 🤖 생성형 AI(Claude) 활용 기록

이 프로젝트는 **Claude (Anthropic의 대화형 AI)** 와 협업해 만들어졌습니다.
사용자는 의뢰·결정·검수를 하고, AI는 코드 작성·디버깅·문서화를 맡았습니다.
각 단계에서 어떻게 분업했는지 기록해둡니다.

### 🤖 단계 1 — 환경 검증
**사용자 의뢰**: "이 폴더에 있는 것들을 검증해주고, 쓰는 방법을 알려줘."

**AI가 한 일**
- 폴더 안의 파일을 직접 열어 구조 파악 (`element_tiles.scad`,
  `generate_tiles.py`, 샘플 CSV)
- 시스템에 OpenSCAD가 설치되어 있는지, 나눔고딕 폰트가 있는지 확인
- 단일 STL 빌드 + 샘플 CSV로 일괄 빌드를 직접 실행해 동작 검증
- Windows 환경에 맞춘 사용법 정리 (PowerShell 명령, OpenSCAD 절대경로 등)

**배운 점** — AI가 단순히 코드를 읽는 데 그치지 않고 실제로 명령을 실행해
보면서 검증할 수 있었기 때문에, "동작 가능" 보고가 추측이 아닌 사실에
기반함.

---

### 🤖 단계 2 — 한글 깨짐 디버깅 (가장 큰 작업)
**사용자 의뢰**: "한글이 깨져서 나오고 있어."

**AI가 한 일**
1. STL을 직접 볼 수 없으니 OpenSCAD로 **PNG 미리보기**를 렌더링하고
   이미지를 직접 확인.
2. "텔루륨" 자리에 두부(☒☒☒) 모양 5개가 보이는 것을 발견.
3. **원인 가설** — Windows의 시스템 코드페이지(CP949)와 UTF-8의 충돌.
4. **검증** — 한글을 UTF-8 파일로 적어 OpenSCAD에 직접 넘기는 실험을 시도
   → 정상 출력됨. 가설 입증.
5. 그 과정에서 **두 번째 함정**(비-ASCII 경로 `include` 실패) 발견 → 임시
   영문 폴더로 우회.
6. `generate_tiles.py` 를 새 방식으로 수정하고 다시 한 번 PNG 렌더로
   "텔루륨" / "철" 이 또렷이 보이는지 시각 검증.

**배운 점** — 텍스트 출력은 깨져도 PNG는 깨지지 않으니, 시각 검증이
인코딩 문제 진단에 결정적이었음. AI가 그래픽 결과를 직접 "보고" 판단할 수
있다는 점이 디버깅에 큰 도움이 됨.

---

### 🤖 단계 3 — 비개발자용 사용 설명서 작성
**사용자 의뢰**: "비개발자를 기준으로 사용방법을 정리한 문서를 만들어줘."

**AI가 한 일**
- "주소창에 powershell 입력해서 검은 창 띄우기" 같은 익숙하지 않을
  단계까지 풀어 설명
- 자주 발생할 오류와 해결법을 별도 섹션으로 정리
- 엑셀에서 CSV 저장할 때 **UTF-8 형식**을 꼭 골라야 한다는 함정 명시

**배운 점** — 개발자에게 "당연한" 단계가 비개발자에게는 가장 큰 장벽.
AI는 자기가 작성한 코드의 사용 흐름을 "처음 보는 사람" 시점으로 다시
재구성하는 데 강함.

---

### 🤖 단계 4 — 폴더 혼동 오류 해결
**사용자 보고**: "`No such file or directory` 오류가 떠."

**AI가 한 일**
- 오류 메시지의 경로(`element-symbol-generator\generate_tiles.py`)와
  실제 파일 위치(`element-symbol-generator\maru-python-version\generate_tiles.py`)를
  비교해 **사용자가 한 단계 위 폴더에서 명령을 실행**한 것을 즉시 추론.
- 즉시 해결 명령(`cd maru-python-version`) 안내 + 사용설명서에 ⚠️ 경고
  추가.

**배운 점** — 오류 메시지의 절대 경로를 파일 시스템 실태와 대조하면
사용자가 "왜 이런 일을" 했는지 빠르게 추론할 수 있다.

---

### 🤖 단계 5 — GUI 설계·구현
**사용자 의뢰**: "비개발자도 쓰기 쉽도록 GUI 형태로 만들어줘. 글자 크기 mm
조절 / 글자 굵기 / 한글 폰트 선택 기능도. 더 추가할 게 있을까?"

**AI가 한 일 (설계 단계)**
- 추가 설치가 필요 없는 **tkinter** 사용 결정 (사용자 대상 고려)
- 4개 탭 구성안 제시 (선택 / 디자인 / 인쇄 / 출력)
- 추가 가능한 기능 6가지를 **장단점과 함께** 제안하고 사용자 선택을 받음

**AI가 한 일 (구현 단계)**
- 118개 원소 데이터 (한국화학회 명명) + 카테고리 + 주기율표 좌표를
  `periodic_table.py` 에 작성
- `generate_tiles.py` 를 **함수 호출형**으로 리팩터링하면서도 기존 CLI
  사용법은 깨지지 않게 유지
- `gui.py` 작성 — 캔버스 기반 주기율표, 백그라운드 스레드 + 진행률,
  설정 저장/불러오기, OpenSCAD 자동 탐지 등
- 더블클릭 실행을 위한 `타일생성기.bat` 추가

**배운 점** — "더 추가할 게 있을까?" 같은 질문에 AI가 곧바로 6가지를
던지면 사용자가 결정 비용이 높음. **장단점**을 함께 제시해야 의사결정이
빨라짐.

---

### 🤖 단계 6 — 화학 테마 디자인
**사용자 의뢰**: "전반적인 디자인도 화학, 원소, 원자 등에 어울리게
만들어줘."

**AI가 한 일**
- **화학 교과서 톤**의 색상 팔레트 결정
  - 배경: 따뜻한 크림색(#F5F1E8 — 양피지)
  - 헤더: 짙은 잉크블루(#1B3A5C)
  - 액센트: 플라스크 틸(#1F6E8C)
- 헤더에 ⚛(원자), 🧪(시험관) 유니코드 기호 사용
- 주기율표 카테고리별 색을 일반 교과서 배색에 맞게 조정
  (알칼리=빨강계열, 할로젠=주황, 비활성기체=보라 등)

**배운 점** — tkinter는 스타일 자유도가 제한적이지만, **색·폰트·여백**만
잘 잡아도 "도구 같지 않은 도구" 느낌이 남.

---

### 🤖 단계 7 — 문서화
**사용자 의뢰**: "(여기까지 한 작업으로) CLAUDE.md 파일을 작성해줘."

**AI가 한 일**
- 첫 버전: 다음 AI 세션이 빠르게 따라잡을 수 있도록 함정·결정·우회 패턴
  중심으로 압축한 기술 메모 작성
- 두 번째 버전(이 문서): **비개발자도 읽을 수 있도록** 풀어 쓰고,
  AI 활용 기록을 별도로 분리

**배운 점** — 같은 정보라도 독자에 따라 구조가 완전히 달라야 함.
"AI 협업 기록"은 별도 섹션으로 분리해야 학교/포트폴리오 자료로 재사용
가능.

## 11. 제공되는 모든 기능 (한눈에)

- **주기율표 GUI** — 118개 원소를 한 번에 보고 클릭으로 선택
- **원소 카테고리 색** — 알칼리/할로젠/비활성기체 등 10개 그룹 색 구분
- **CSV 입력 모드** — 자기가 만든 표 파일도 사용 가능
- **글자 디자인** — 4개 텍스트 각각의 mm 크기, 폰트, 굵기, 추가 두껍게,
  새김 깊이
- **상용 가능 한글 폰트만** 자동 필터해 드롭다운에 표시
- **인쇄 설정** — 타일 크기, 두께, 옆 타일과의 끼움 여유
- **OpenSCAD 자동 탐지** — 처음 한 번만 자동 탐지 클릭
- **결과 폴더 자동 열기 / 완료 알림음**
- **프리셋 저장·불러오기** — JSON 파일로 설정 묶음 보관
- **마지막 사용 설정 자동 기억** — 다음에 켜면 동일한 상태로 시작
- **백그라운드 작업 + 진행률 + 중단** — 큰 일괄 생성 중에도 GUI 안 멈춤
- **중학교 권장 15종 빠른 선택** 버튼

## 15. 기술 메모 (개발자가 코드를 만질 때)

> 이 절은 비개발자는 읽지 않아도 됩니다. 코드를 수정할 사람만 참고.

- 새 SCAD 파라미터 추가 시 **세 곳**을 동시에 수정해야 함
  - `element_tiles.scad` 변수 선언
  - `generate_tiles.py` 의 `TileParams` dataclass + `to_scad_lines()`
  - `gui.py` 의 `_setup_vars` / 위젯 / `_collect_config` / `_apply_config`
- GUI 위젯 접근은 메인 스레드만. OpenSCAD 호출은 워커 스레드 → `Queue`
  → `root.after()` 폴링.
- 시각 검증은 `render_preview_png()` 헬퍼로. 카메라 옵션
  `--autocenter --viewall --projection=ortho` 가 기본 권장.
- 설정/프리셋 위치: `~/.element_tile_generator/config.json`,
  `~/.element_tile_generator/presets/*.json`
- OpenSCAD의 `text()` 는 폰트 굵기 변형이 없으면 자동으로 Regular로
  fallback. UI에서 ExtraBold 선택했는데 폰트가 그 변형을 안 가지고 있으면
  사용자에게 알리지 않으니 **`text_thicken` 으로 보완**하는 안내가 필요.
